API Documentation

All endpoints return JSON. Base path: /api/v1. No authentication required.

Response Envelope

Every response wraps its payload in a top-level key.

Success

{
  "data": { ... } | [ ... ],
  "items": <int>         // count, present on list responses
}

Error

{
  "error": {
    "status_code": 400,
    "message": "Human-readable description."
  }
}

Stations

GET /api/v1/stations

Returns a paginated list of all weather stations.

ParameterInTypeRequiredDescription
pagequeryintegerNoPage number (default: 1). Returns {"data": []} if beyond last page.

Example request

GET /api/v1/stations?page=2

Success response 200

{
  "data": [
    { "STAID": "000001", "STANAME": "VILNIUS" },
    { "STAID": "000002", "STANAME": "TALLINN" }
  ],
  "items": 97,
  "page": 2,
  "page_size": 20,
  "total_pages": 5,
  "has_next": true
}
GET /api/v1/stations/search

Returns up to 50 stations whose names contain the query string (case-insensitive).

ParameterInTypeRequiredDescription
namequerystringYesPartial name to search for. Returns 400 if missing or empty.

Example request

GET /api/v1/stations/search?name=vil

Success response 200

{
  "data": [
    { "STAID": "000001", "STANAME": "VILNIUS" }
  ],
  "search_query": "vil",
  "items": 1
}

Error response 400

{ "error": { "status_code": 400, "message": "No value for param 'name' is provided" } }

Station Temperature Data

GET /api/v1/station/{id}?date=YYYY-MM-DD

Returns the recorded temperature for a single date at the given station. Provide exactly one of date or year.

ParameterInTypeRequiredDescription
idpathintegerYesNumeric station ID (e.g. 1, 199).
datequerystringYes*Date in YYYY-MM-DD format. Mutually exclusive with year.

Example request

GET /api/v1/station/1?date=2003-08-12

Success response 200

{
  "data": {
    "stationid": "000001",
    "date": "2003-08-12",
    "temperature": 38.2
  },
  "items": 1
}

Error responses

// 400 — invalid date format
{ "error": { "status_code": 400, "message": "Invalid date format. Expected YYYY-MM-DD." } }

// 400 — both date and year supplied
{ "error": { "status_code": 400, "message": "Conflicting parameters: provide either 'date' or 'year', not both." } }

// 404 — station file not found or no record for date
{ "error": { "status_code": 404, "message": "Station not found." } }
GET /api/v1/station/{id}?year=YYYY

Returns all daily temperature records for a station for a given year. Missing values (-9999) are returned as null.

ParameterInTypeRequiredDescription
idpathintegerYesNumeric station ID.
yearqueryintegerYes*Four-digit year (e.g. 2003). Mutually exclusive with date.

Example request

GET /api/v1/station/1?year=2003

Success response 200

{
  "data": [
    { "date": "2003-01-01", "temperature": -3.1 },
    { "date": "2003-01-02", "temperature": null },
    { "date": "2003-12-31", "temperature": 1.4 }
  ],
  "items": 365
}

Error responses

// 400 — invalid year value
{ "error": { "status_code": 400, "message": "Invalid year." } }

// 404 — station file not found
{ "error": { "status_code": 404, "message": "Station not found." } }
GET /api/v1/station/{id}/yearly

Returns one mean temperature value per year for the station, aggregated from all valid daily records. Years with no valid data are omitted.

ParameterInTypeRequiredDescription
idpathintegerYesNumeric station ID.

Example request

GET /api/v1/station/1/yearly

Success response 200

{
  "data": [
    { "year": 1960, "temperature": 6.1 },
    { "year": 1961, "temperature": 5.8 }
  ],
  "items": 62,
  "stationid": "000001"
}

Error response 404

{ "error": { "status_code": 404, "message": "Station not found." } }
GET /api/v1/station/{id}/date/{mm_dd}

Returns the temperature recorded on a specific calendar date (MM-DD) across all available years.

ParameterInTypeRequiredDescription
idpathintegerYesNumeric station ID.
mm_ddpathstringYesCalendar date in MM-DD format (e.g. 07-15).

Example request

GET /api/v1/station/1/date/07-15

Success response 200

{
  "data": [
    { "date": "2000-07-15", "temperature": 24.3 },
    { "date": "2001-07-15", "temperature": 21.7 }
  ],
  "items": 60,
  "stationid": "000001",
  "mm_dd": "07-15"
}

Insights

GET /api/v1/insights/{id}?type=<type>

Returns a statistical insight for the given station. The date parameter is required only for date-dependent insight types.

ParameterInTypeRequiredDescription
idpathintegerYesNumeric station ID.
typequerystringYesInsight type — see table below.
datequerystringConditionalCalendar date in MM-DD format. Required for avg_for_date and temp_variability.

Insight types

type valuedate requiredResponse data shapeDescription
hottest_yearNo{"year": int, "value": float}Year with highest mean daily temperature and its value.
coldest_yearNo{"year": int, "value": float}Year with lowest mean daily temperature and its value.
hottest_dayNo{"date": "YYYY-MM-DD", "value": float}Single hottest recorded date and temperature in °C.
coldest_dayNo{"date": "YYYY-MM-DD", "value": float}Single coldest recorded date and temperature in °C.
avg_for_dateYes (MM-DD){"avg_temp": float}Mean temperature across all years for that calendar date, rounded to 1 d.p.
temp_variabilityYes (MM-DD){"std_dev": float}Standard deviation of temperature across all years for that calendar date, rounded to 1 d.p.
missing_data_countNo{"missing_count": int}Count of missing (-9999) temperature records in the station file.

Example requests

GET /api/v1/insights/1?type=hottest_year
GET /api/v1/insights/1?type=avg_for_date&date=07-15

Success responses 200

// hottest_year / coldest_year
{ "data": { "year": 2003, "value": 12.4 } }

// hottest_day / coldest_day
{ "data": { "date": "2003-08-12", "value": 38.2 } }

// avg_for_date
{ "data": { "avg_temp": 22.4 } }

// temp_variability
{ "data": { "std_dev": 3.1 } }

// missing_data_count
{ "data": { "missing_count": 42 } }

Error responses

// 400 — unsupported type
{ "error": { "status_code": 400, "message": "Invalid insight type." } }

// 400 — date missing for date-dependent insight
{ "error": { "status_code": 400, "message": "Parameter 'date' (MM-DD) is required for this insight type." } }

// 404 — station not found
{ "error": { "status_code": 404, "message": "Station not found." } }

Compare

GET /api/v1/compare

Returns a full-year union of daily temperatures for two stations side by side. Every calendar day in the requested year is present; missing values are null.

ParameterInTypeRequiredDescription
stationA_idqueryintegerYesNumeric station ID for Station A.
stationB_idqueryintegerYesNumeric station ID for Station B.
yearqueryintegerYesFour-digit year to compare (e.g. 2003).

Example request

GET /api/v1/compare?stationA_id=1&stationB_id=2&year=2003

Success response 200

{
  "data": [
    { "date": "2003-01-01", "stationA": -3.1, "stationB": -1.8 },
    { "date": "2003-01-02", "stationA": null, "stationB": -0.4 },
    { "date": "2003-12-31", "stationA": 1.4,  "stationB": null  }
  ],
  "items": 365,
  "stationA_id": "000001",
  "stationB_id": "000002",
  "year": "2003"
}

Error responses

// 400 — missing required parameter
{ "error": { "status_code": 400, "message": "Parameter 'stationA_id' is required." } }

// 400 — invalid year or non-numeric station ID
{ "error": { "status_code": 400, "message": "Invalid year." } }

// 404 — station file not found
{ "error": { "status_code": 404, "message": "Station not found." } }
